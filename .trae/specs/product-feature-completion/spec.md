# MediMate 商用品化功能补全计划

## 为什么
MediMate 项目当前处于功能原型阶段，虽然核心 UI 已完成，但存在大量功能缺失，无法作为正式商用产品投入市场。主要问题包括：用户无法注册账号、个人资料无法编辑、支付流程不完整、消息系统功能残缺、后台管理功能不完善等。

## 什么变化
- 实现完整的用户认证系统（注册、登录、密码找回）
- 实现个人资料管理（头像上传、资料编辑）
- 完善患者端下单流程
- 完善陪诊师端接单和订单管理
- 实现完整的支付功能（微信支付、退款）
- 完善消息系统（搜索、发起对话、媒体发送）
- 完善通知系统（点击跳转、偏好设置）
- 完善后台管理系统（编辑、导出、退款管理）
- 实现通用功能（深色模式、清除缓存、版本更新）

## 影响
- 受影响的规范：用户认证、个人资料、订单管理、支付系统、消息系统、通知系统、后台管理
- 受影响的代码：components/ 目录、services/apiService.ts、App.tsx、server/src/modules/

## 新增需求

### 需求：用户注册与认证系统
系统应提供完整的用户注册和认证功能

#### 场景：用户注册
- **当** 用户点击注册入口时
- **则** 显示注册表单，包含邮箱、密码、确认密码、角色选择
- **当** 用户填写信息并提交时
- **则** 验证邮箱格式和密码强度，创建用户账号，跳转到登录页

#### 场景：忘记密码
- **当** 用户点击忘记密码时
- **则** 显示邮箱输入框，发送重置链接到邮箱

#### 场景：登录状态持久化
- **当** 用户登录成功时
- **则** 保存 token 到 localStorage，刷新页面后自动恢复登录状态

### 需求：个人资料管理
系统应允许用户编辑和管理个人资料

#### 场景：上传头像
- **当** 用户点击头像上传区域时
- **则** 打开文件选择器，支持 jpg/png/webp 格式，最大 2MB
- **当** 用户选择图片并确认时
- **则** 上传到服务器，更新页面显示新头像

#### 场景：编辑资料
- **当** 用户点击编辑资料按钮时
- **则** 显示编辑表单，允许修改姓名、手机号、简介
- **当** 用户提交修改时
- **则** 调用 API 保存，显示保存成功提示

### 需求：患者端下单流程
系统应支持患者完成完整的陪诊服务下单

#### 场景：发起陪诊请求
- **当** 患者选择陪诊师并点击下单时
- **则** 显示订单确认页面，包含服务详情、价格、时间
- **当** 患者确认并完成支付时
- **则** 创建订单，发送通知给陪诊师，显示订单详情

#### 场景：查看订单详情
- **当** 患者点击订单列表中的订单时
- **则** 显示订单详情页面，包含陪诊师信息、服务状态、聊天入口

### 需求：陪诊师端订单管理
系统应支持陪诊师完成订单服务

#### 场景：完成陪诊服务
- **当** 陪诊师完成服务后点击完成按钮时
- **则** 弹出确认框，确认后更新订单状态为已完成，触发患者评价

#### 场景：查看收入明细
- **当** 陪诊师查看收入时
- **则** 显示收入明细列表，包含每笔收入的时间、订单号、金额

#### 场景：申请提现
- **当** 陪诊师点击提现按钮时
- **则** 显示提现表单，填写金额和支付宝/微信账号，提交申请

### 需求：支付功能完整性
系统应支持完整的支付流程

#### 场景：微信支付
- **当** 用户选择微信支付时
- **则** 显示微信支付二维码，用户扫码支付后自动跳转成功页面

#### 场景：申请退款
- **当** 用户在订单详情页点击退款时
- **则** 显示退款原因表单，提交后等待审核，退款审核通过后原路返回

### 需求：消息系统完善
系统应提供完整的即时通讯功能

#### 场景：发起新对话
- **当** 用户点击新消息按钮时
- **则** 显示联系人列表或搜索框，用户选择后创建新对话

#### 场景：搜索消息
- **当** 用户在搜索框输入关键词时
- **则** 实时搜索消息内容，高亮显示匹配结果

#### 场景：发送媒体消息
- **当** 用户点击附件按钮时
- **则** 打开文件选择器，支持图片/文件，发送后显示预览

### 需求：通知系统完善
系统应提供可交互的通知功能

#### 场景：通知点击跳转
- **当** 用户点击订单相关通知时
- **则** 跳转到订单详情页
- **当** 用户点击消息通知时
- **则** 跳转到聊天页面

#### 场景：设置通知偏好
- **当** 用户进入通知设置时
- **则** 可以开关各类型通知（订单、消息、系统）

### 需求：后台管理功能完善
系统应提供完整的后台管理功能

#### 场景：编辑用户资料
- **当** 管理员在用户列表点击编辑时
- **则** 显示编辑表单，可以修改用户信息

#### 场景：处理退款申请
- **当** 管理员进入退款管理时
- **则** 显示退款列表，可以批准或拒绝退款申请

#### 场景：导出数据
- **当** 管理员点击导出按钮时
- **则** 导出用户/订单数据为 Excel/CSV 文件

### 需求：通用功能
系统应提供设置等通用功能

#### 场景：深色模式切换
- **当** 用户在设置中点击深色模式时
- **则** 切换应用主题，保存偏好到 localStorage

#### 场景：清除缓存
- **当** 用户点击清除缓存时
- **则** 清除本地存储的临时数据，显示清理成功提示

## 修改需求

### 需求：Header 组件优化
**修改内容**：动态显示用户头像和未读消息数

- 头像应从用户资料 API 获取
- 通知图标应显示未读数量徽章
- 消息图标应显示未读数量徽章

### 需求：Profile 组件功能实现
**修改内容**：实现所有占位功能的实际功能

- 头像上传功能
- 编辑资料功能
- 标签页内容实际加载

## 移除需求
- 无
